## Color Accessibility Notes

### Contrast Ratios (WCAG AA)
- All primary colors used with white: ✅ Passes
- All secondary colors used with dark text: ✅ Passes
- Accent colors on backgrounds: ⚠️ Test specific combinations

### Color-Blind Considerations
- Purple/Blue: ❌ May be confused
- Red/Orange: ⚠️ Deuteranopia issues
- Solution: Use distinct shapes + color

### Recommended Backgrounds
- Glimmer Purple: Light background (`#F3E8FF`)
- Pixel Blue: Light background (`#EFF6FF`)
- Zing Yellow: Dark background (`#1F2937`)
- Boom Red: Light background (`#FEE2E2`)

## Animation Performance Notes

### GPU-Accelerated Animations
These animations perform best:
- `transform` based: bounce, spin, translate
- `opacity` changes: fade_in, shimmer
- `filter` changes: glow, blur

### CPU-Heavy Animations
Use sparingly or optimize:
- Complex `box-shadow` (glow alternatives)
- Multiple simultaneous animations
- Animations on mobile devices

### Optimization Tips
1. Use `will-change: transform` for animated elements
2. Reduce motion for users with `prefers-reduced-motion`
3. Pause animations when off-screen
4. Cache animation styles
5. Use `transform: translateZ(0)` for hardware acceleration

## For Visual Design

### Brand Consistency
All creatures maintain these design principles:
- Rounded, friendly shapes (no sharp edges)
- Large expressive eyes
- Warm, inviting color palettes
- Clear emotional expression
- Consistent stroke weight (~2-3px)

### Style Guide
- Vector format (SVG recommended)
- 256px base size for all creatures
- Consistent proportions (roughly square or rectangular)
- Eyes: 40-50% of head size
- Expressions through eye/mouth changes primarily

### Animation Art Principles
- Anticipation: Small setup before main action
- Follow-through: Slight trailing motion
- Staging: Make intent clear immediately
- Easing: Avoid linear motion
